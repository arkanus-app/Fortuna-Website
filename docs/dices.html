Dice Syntax

We use the standard dice syntax for RPG, <dice count> d <dice faces count>

Exemples:
5d20: Rolls 5 20-sided dice and returns the sum of all of them
100d4: Rolls 100 4-sided dice and returns the sum of all of them
By default, if the number of dice is not written, it will be considered 1, and if the number of sides is not written, it will be considered 20.
Exemples
d5: Roll 1 5-sided dices
42d: Roll 42 20-sided dices
d: Roda 1 20-sided dice
Note: To avoid DoS attacks on the bot, the total amount of dice cannot exceed 200, and the maximum number of sides of a dice is 1000.

But our bot has way more modifiers than just ordinary RPG dice!

Every modifier will be written as follows: <modifier>d<number>
Where modifier is the letter (or letters) that represents it, and number the number that this modifier receives.
All data modifiers are combinable with each other, unless otherwise specified.

Explosive

Explosive dice are dice that, if the roll drops a certain number or more, roll again.
Syntax:
e: If the rolled number is greater than the requested number, the die explodes only once.
ei: Explode until the result is less than the requested number.
Exemples:
d20e5: If the original dice lands greater than or equal to 5, roll one more time.
30ei10: If the die lands greater than or equal to 10, it rolls again, and if the new die also rolls again, until the result is less than 10
By default, if the number is not written, the maximum number of the data will be considered.
Examples:
15d24e = 15d24e245d4
ei = 5d4ei4
Note: To avoid DoS attacks on the bot, we limit the number of explosions to 200, since using it legitimately you will never need more than that.

Interval

Only numbers in the given range are kept.
Syntax:
<<: Keep only the numbers less than or equal to the requested one, the rest are discarded.

>>: Keep only the numbers greater than or equal to the requested one, the rest are discarded.

Exemples:
12d30<<25: Keep only the numbers less than or equal to 25, the rest discard.
12d10>>5: Keep only the numbers greater than or equal to 5, the rest discard.
12d50<<35>>10: Keep only numbers between 10 and 35, discard the rest.
Keep only numbers between 10 and 35, discard the rest.
Exemples:
6d32>> = 6d32>>16
8d42<< = 8d42<<21

Keep

Only the largest or smallest numbers are kept.
Syntax:
k: Keep only the highest numbers on the roll, discard the rest.
km: Keep only the lowest numbers on the roll, discard the rest.
Exemples:
12dk5: Assuming a result of{3, 5, 10, 15, 2, 1, 6, 8, 2, 20, 9, 19}, will keep only the top 5, i.e., {10, 15, 20, 9, 19}

12dkm5: Assuming the same result as before ({3, 5, 10, 15, 2, 1, 6, 8, 2, 20, 9, 19}), will keep only the 5 smallest ones, i.e., {3, 5, 2, 1, 2}

12dkm2k3: Assuming the same result as before ({3, 5, 10, 15, 2, 1, 6, 8, 2, 20, 9, 19}), will keep only the 2 smallest and the 3 largest i.e. {15, 2, 1, 20, 19}By default, if no number is written, it will be assumed to be 1.
Exemples
15d24k = 15d24k15d4km = 5d4km1

Bulk operators

Bulk operator expressions work from left to right, regardless of the math priority order. This is a design choice to avoid complicating the data by having to put parentheses around the data.
Syntax:
++: Adds the given number to all the dice.
--: Subtracts the given number from all dice.
**: Multiplica todos os dados pelo número determinado.
//: Multiplies all dice by the given number.
%%: Calculates the remainder of division by the given number. (Honestly, who's going to use this in their life?)

Exemples:
23d22++6: Add 6 to each of the dice.
2d2--4**8: Subtract 4 and then multiply each dice by 8.
2d2**8--4: Multiply by 8 and then subtract 4 on each of the dice.

Fate Dices

These are data that only have 3 possible values: -1, 0 or 1.
And yes, that's all they do.
The Fate system uses them, so yeah, there you go.
f<número>
Default: 4

Exemples
d+f: Roll d20 + 4 fate dices.
f10: Roll 10 fate dices.



Numerical Expressions

And of course, numerical expressions could not be missing!
These work just like you learned in school.

The expression (5 + 10 * 8) / (10 - 15) will result in -17.
But of course, our data is not just a calculator.

Allowed Operators:
+: Sum.
-: Subtract.
*: Multiply.
/: Divide.
%: Calculates the remainder of the division. (I still want to know who will use this in life)
(): Changes the order of origin of the expression

Exemples
23d20+3: Add 3 to the dice result.

5d3+33d4*3d10: Multiply the result of 3d10 by the result of 33d4, then add to the result of 5d3.
