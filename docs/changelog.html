1.10.0 - (And probably the biggest update for this bot to date)

+ Added die modifiers << and >>
They aim to keep only numbers less than or equal to (<<) or greater than or equal to (>>) than the number to their right.
E.g.: 10d20<<10 => will roll 10 20-sided dice and keep only rolls less than or equal to 10

+ Now k and km data modifiers work together
Eg: 10d20 k3 km5 => will keep the 5 smallest and 3 largest, the rest will be discarded

+ The bot was migrated to a new host, with more space, processing and a MySQL database
+ We took advantage of the new host and migrated all the bot's save (which was a .json file) to this database
+ Premium servers are finally working, and premium members now don't necessarily need to be on this server anymore. We have updated /donate to pass the new information.
+ The /di and /ficha commands (which now have an alias in English, /card) were practically rebuilt from scratch, now prepared to receive the update from different RPG systems (probably will come out in the next update)
+ Added the /update command, which sends the latest bot update
+ A website was made for the bot, with some information about it, such as a tutorial on how to use it, terms of use, team presentation and information on how to donate. The website link is [https://sites.google.com/view/minikraken](Yes, it's google sites because we don't have money for our own domain) (the old non-old website doesn't exist anymore, after all this one is very better)

1.9.0

+ Added "keep minimum" data (km):
5d20km2 => Will roll 5 dice from 20 sides and keep the two smallest values.

+ Added explosive dice
d10e10 => Will roll a 10-sided die, and will roll again if the result is greater than or equal to 10.

+ Added explosive dice indefinitely
d10ei10 => Rolls a 10-sided die, explodes in 10 and if the value is again greater than or equal to 10, it explodes again. It will keep exploding until the value is less than 10.
(Limited to 200 explosions per command to prevent abuse.)
NOTE: You cannot use explosive dice and "keep" dice at the same time.

+ Added support for math expressions
Now you can write d20+6 / (4d20 + d20k2) for example. (Limited to addition, subtraction, multiplication, division and remainder)

+ Added some writing shortcuts
d = 1d20: A die without the number of sides will be understood as a 20-sided die;
3dk = 3d20k1: A keep die with no number in front will be understood as "keep one" (the same applies to km data);
5d10e = 5d10e10: An explosive die with no explosion number will be understood to "explode at the highest die value" (same applies to ei).
